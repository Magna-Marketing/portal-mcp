Tool: list_promotions

Description
List `Promotion` records (program-wide promotions, partner promotions,
tiered promotions). Filter by program, status, channel, target, market
segment, date window. For per-promotion drill-down with rules use
`get_promotion_details`.

Salesforce API
REST API (Data), v66.0. SOQL on `Promotion`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, Name, Description, ProgramId, Program.Name, PromotionType, Status, EffectiveStartDate, EffectiveEndDate, ChannelId, Stage, IsActive, EnrollmentRequired, MaxMembershipCount, EnrolledMembersCount, CreatedDate, LastModifiedDate FROM Promotion

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
programName (string, optional).
programId (string, optional).
status (string|string[], optional) - "Active" | "Inactive" | "Draft" | "Expired".
promotionType (string|string[], optional).
channelName (string, optional).
isActive (boolean, optional).
namePattern (string, optional).
limit (number, optional, default 200, max 2000).

Result
totalSize (number).
promotions (array) - { id, name, description, programId, programName, promotionType, status, effectiveStartDate, effectiveEndDate, channelId, stage, isActive, enrollmentRequired, maxMembershipCount, enrolledMembersCount, createdDate, lastModifiedDate }.

Required permissions
- `Loyalty Management` permission set.

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_promotions({ "programName": "MTG_Rewards", "status": "Active" })
