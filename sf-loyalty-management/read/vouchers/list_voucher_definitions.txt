Tool: list_voucher_definitions

Description
List `VoucherDefinition` records: the templates used to issue vouchers
(value, validity period, partner restrictions, redemption channel,
status). Pair with `issue_voucher` to mint a voucher for a member.

Salesforce API
REST API (Data), v66.0. SOQL on `VoucherDefinition`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, Name, Description, ProgramId, Program.Name, Type, ValueType, FaceValue, DiscountPercent, ValidityType, ValidityDuration, ValidityStartDate, ValidityEndDate, IsActive, MaxAllowedRedemptions, RedeemedQuantity, IssuanceCriteria, PartnerAccountId, IsTransferable, ReferralVoucherId, AddOnSourceId, CreatedDate FROM VoucherDefinition

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
programName (string, optional).
programId (string, optional).
type (string, optional) - "FixedValue" | "Discount" | "ProductOrService".
valueType (string, optional) - "Currency" | "Percent" | "Other".
isActive (boolean, optional, default true).
namePattern (string, optional).
partnerAccountId (string, optional).
limit (number, optional, default 200, max 2000).

Result
totalSize (number).
voucherDefinitions (array) - { id, name, description, programId, programName, type, valueType, faceValue, discountPercent, validityType, validityDuration, validityStartDate, validityEndDate, isActive, maxAllowedRedemptions, redeemedQuantity, partnerAccountId, isTransferable, createdDate }.

Required permissions
- `Loyalty Management` permission set.

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_voucher_definitions({ "programName": "MTG_Rewards", "isActive": true })
