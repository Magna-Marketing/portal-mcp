Tool: list_member_benefits

Description
List `MemberBenefit` records — benefits granted directly to a member
(outside the tier-driven path). Used for one-off rewards or
admin-issued perks.

Salesforce API
REST API (Data), v66.0. SOQL on `MemberBenefit`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, MemberId, Member.MembershipNumber, BenefitId, Benefit.Name, Benefit.BenefitType.Name, IssuedDate, EffectiveStartDate, EffectiveEndDate, IsActive, RelatedTransactionJournalId, CreatedDate FROM MemberBenefit

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
memberId (string, optional).
membershipNumber (string, optional).
benefitId (string, optional).
isActive (boolean, optional, default true).
limit (number, optional, default 200).

Result
totalSize (number).
memberBenefits (array) - { id, memberId, membershipNumber, benefitId, benefitName, benefitTypeName, issuedDate, effectiveStartDate, effectiveEndDate, isActive, relatedTransactionJournalId, createdDate }.

Required permissions
- `Loyalty Management` permission set.

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_member_benefits({ "memberId": "0lMQg00000abcDE2", "isActive": true })
