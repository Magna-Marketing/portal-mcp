Tool: list_game_participants

Description
List `GameParticipant` rows — each entry represents a member's
participation in a given game (one row per playthrough). Includes the
reward earned and timestamp. Useful for game telemetry and member-level
audits.

Salesforce API
REST API (Data), v66.0. SOQL on `GameParticipant` and
`GameParticipantReward`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, GameDefinitionId, GameDefinition.Name, MemberId, Member.MembershipNumber, ParticipationDate, ParticipationStatus, RewardId, Reward.Name, RewardType, ParticipationChannel, CreatedDate FROM GameParticipant

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
gameDefinitionId (string, optional).
gameName (string, optional).
memberId (string, optional).
sinceDays (number, optional, default 30).
participationStatus (string, optional) - "Won" | "Lost" | "Pending" | "Skipped".
limit (number, optional, default 200).

Result
totalSize (number).
participants (array) - { id, gameDefinitionId, gameName, memberId, membershipNumber, participationDate, participationStatus, rewardId, rewardName, rewardType, participationChannel, createdDate }.

Required permissions
- `Loyalty Management` permission set.

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_game_participants({ "gameName": "Spin_The_Wheel_Q3", "sinceDays": 7 })
