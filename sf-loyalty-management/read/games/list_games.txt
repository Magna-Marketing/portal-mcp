Tool: list_games

Description
List `GameDefinition` records (Loyalty Gamification): the playable
games members can engage with (Spin the Wheel, Scratch Card, Pick One,
Quiz, etc.). Returns identity, status, mechanics, eligibility window.

Salesforce API
REST API (Data), v66.0. SOQL on `GameDefinition`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, Name, Description, GameType, ProgramId, Program.Name, Status, EffectiveStartDate, EffectiveEndDate, MaxParticipationCount, IsActive, ImageUrl, RuleSet, CreatedDate FROM GameDefinition

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
programName (string, optional).
programId (string, optional).
gameType (string, optional) - "SpinTheWheel" | "ScratchCard" | "PickOne" | "Quiz" | "RevealReward" | etc.
status (string, optional) - "Active" | "Inactive" | "Draft".
isActive (boolean, optional).
namePattern (string, optional).
limit (number, optional, default 200).

Result
totalSize (number).
games (array) - { id, name, description, gameType, programId, programName, status, effectiveStartDate, effectiveEndDate, maxParticipationCount, isActive, imageUrl, ruleSet, createdDate }.

Required permissions
- `Loyalty Management` permission set.

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_games({ "programName": "MTG_Rewards", "isActive": true })
