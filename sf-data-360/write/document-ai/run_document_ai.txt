Tool: run_document_ai

Description
Trigger a Document AI processing run for a configuration. Async;
the platform processes pending documents (or a supplied batch) and
ingests the extracted records into the target DMO.

Salesforce API
Connect plane.

Endpoint
POST {instanceUrl}/services/data/v{ver}/ssot/document-processing/configurations/:idOrApiName/actions/run

Headers
Authorization: Bearer <core_access_token>
Content-Type: application/json
Accept: application/json

Body
{
  "documentRefs": ["s3://magnatech-docs/invoices/2026-04/"],
  "runMode": "Batch"
}

Input (parameters)
idOrApiName (string, required).
documentRefs (string[], optional) - URIs to process. Omit to process
  the configuration's default queue.
runMode (string, optional, default "Batch") - "Batch" | "OnDemand".
runIdempotencyKey (string, optional).

Result
runId (string).
configurationApiName (string).
status (string) - "Queued".
submittedAt (ISO-8601).
documentCount (number).

Required scopes
- `api`.

Idempotency & limits
- Idempotent on `runIdempotencyKey`.
- Subject to org-level Document AI quota.

Errors
- 404 NOT_FOUND, 409 JOB_IN_PROGRESS, 400 INVALID_DOCUMENT_REFS.

Example request

run_document_ai({
  "idOrApiName": "Invoice_Default",
  "documentRefs": ["s3://magnatech-docs/invoices/2026-04/inv_001.pdf"]
})

Example result
{
  "runId": "RUN_DOC_2026-04-29T13",
  "configurationApiName": "Invoice_Default",
  "status": "Queued",
  "documentCount": 1
}
