Tool: get_profile_metadata

Description
List the Profile-eligible DMOs (DMOs whose `category=Profile`) along
with their searchable identifiers and relationships. Used as a
discovery endpoint before calling `get_profile_record`.

Salesforce API
Connect plane (default) or runtime plane.

Endpoint
GET {instanceUrl}/services/data/v{ver}/ssot/profile/metadata           (plane="connect", default)
GET {cdpTenantUrl}/api/v1/profile/metadata                             (plane="runtime")

Headers
Authorization: Bearer <core_access_token>      (connect)
Authorization: Bearer <data_cloud_token>       (runtime)
Accept: application/json

Body
None.

Input (parameters)
plane (string, optional, default "connect").
dataSpace (string, optional, default "default").

Result
profileObjects (array) - Each: { dmoApiName, label, primaryKeyField,
  searchableFields (array), relationships (array<{ name, toDmo }>) }.

Required scopes
- `api` (connect) or `cdp_profile_api_read` (runtime).

Idempotency & limits
- Read-only.

Errors
- 401 INVALID_SESSION_ID.

Example request

get_profile_metadata({})

Example result
{
  "profileObjects": [
    { "dmoApiName": "UnifiedIndividual__dlm",
      "primaryKeyField": "Id__c",
      "searchableFields": ["Id__c","EmailAddress__c"],
      "relationships": [
        { "name": "EngagementHistory", "toDmo": "EngagementHistory__dlm" }
      ] }
  ]
}
