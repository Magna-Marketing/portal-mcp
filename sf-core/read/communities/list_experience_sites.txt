Tool: list_experience_sites

Description
List Experience Cloud sites (`Network` sObject) — public communities,
partner portals, customer portals. Returns identity, status, URL,
template, login policy, and member count.

Salesforce API
REST API + Connect API, v66.0.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, Name, UrlPathPrefix, Status, OptionsHttps, Description, CreatedDate FROM Network
GET /services/data/v66.0/connect/communities                                  (richer Connect API view)

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
status (string, optional) - "Live" | "DownForMaintenance" | "UnderConstruction".
namePattern (string, optional).
includeMemberCount (boolean, optional, default true).

Result
totalSize (number).
sites (array) - { id, name, urlPathPrefix, status, siteUrl, optionsHttps, description, template (string), createdDate, memberCount (when includeMemberCount=true) }.

Required permissions
- "Manage Experiences" / "Create and Set Up Experiences".

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_experience_sites({ "status": "Live" })
