Tool: view_experience_site

Description
Get a single Experience site's full configuration: navigation, branding
sets, audiences, member profiles allowed, login flows, page snapshots.

Salesforce API
Connect API, v66.0. `/connect/communities/{communityId}`.

Endpoint
GET /services/data/v66.0/connect/communities/{communityId}
GET /services/data/v66.0/connect/communities/{communityId}/membership

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
siteId (string, optional) - Network Id.
siteName (string, optional).
  One required.

Result
site (object) - { id, name, description, urlPathPrefix, status, siteUrl, template, allowMembersToFlag, contentVisibility }.
membership (object) - { profiles: [{ profileId, name }], permissionSets: [{ id, name }], hasAccessToInternalUsers (boolean) }.

Required permissions
- "Create and Set Up Experiences".

Idempotency & limits
- Read-only.

Errors
- NOT_FOUND.

Example request

view_experience_site({ "siteName": "Customer_Hub" })
