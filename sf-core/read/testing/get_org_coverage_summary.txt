Tool: get_org_coverage_summary

Description
Get the org-wide Apex code coverage percent (the number Salesforce
checks against the 75% production-deploy threshold). Also returns
breakdown by class type (controllers, services, batch, triggers, test
helpers).

Salesforce API
Tooling API, v66.0. Aggregates `ApexCodeCoverageAggregate` and
`ApexOrgWideCoverage`.

Endpoint
GET /services/data/v66.0/tooling/query/?q=SELECT PercentCovered FROM ApexOrgWideCoverage
GET /services/data/v66.0/tooling/query/?q=SELECT ApexClassOrTrigger.Name, NumLinesCovered, NumLinesUncovered FROM ApexCodeCoverageAggregate

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
breakdown (boolean, optional, default true) - Include per-class breakdown.
underThreshold (number, optional) - Filter `breakdown` to classes with coverage below this percent (e.g. 75).

Result
orgWidePercent (number) - 0..100.
totalLinesCovered (number).
totalLinesUncovered (number).
classCount (number).
triggerCount (number).
breakdown (array, when breakdown=true) - { name, type, numLinesCovered, numLinesUncovered, percentCovered }.
classesUnderThreshold (number, when underThreshold provided).

Required permissions
- "View All Data" / "Author Apex".

Idempotency & limits
- Read-only.
- Recompute by re-running tests.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

get_org_coverage_summary({ "underThreshold": 75 })
