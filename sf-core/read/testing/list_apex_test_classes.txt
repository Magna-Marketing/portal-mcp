Tool: list_apex_test_classes

Description
List Apex classes that contain unit tests (annotated with `@isTest` or
`testMethod`). Returns class identity plus the count of test methods
inside. Pair with `run_apex_tests` to execute, or `get_code_coverage`
to see what they cover.

Salesforce API
Tooling API, v66.0. SOQL on `ApexClass WHERE Body LIKE '%@isTest%'`,
or use the symbol-table approach for accuracy.

Endpoint
GET /services/data/v66.0/tooling/query/?q=SELECT Id, Name, NamespacePrefix, ApiVersion, Status, IsValid, SymbolTable FROM ApexClass WHERE Body LIKE '%@isTest%' OR Body LIKE '%testMethod%'

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
namePattern (string, optional).
namespacePrefix (string, optional).
managedOnly (boolean, optional, default false).
includeMethodCounts (boolean, optional, default true) - Use SymbolTable to count test methods per class.
limit (number, optional, default 500).

Result
totalSize (number).
testClasses (array) - { id, name, namespacePrefix, apiVersion, isValid, status, testMethodCount (when includeMethodCounts) }.

Required permissions
- "View All Data" / "Author Apex".

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_apex_test_classes({ "namePattern": "*Test", "includeMethodCounts": true })
