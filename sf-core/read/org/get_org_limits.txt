Tool: get_org_limits

Description
Get org-wide REST API limits and current usage: DailyApiRequests,
DataStorageMB, FileStorageMB, DailyAsyncApexExecutions,
DailyBulkApiBatches, ConcurrentSyncLongRunningApexExecutions, etc.
Use to monitor headroom before bulk loads or scheduled jobs.

Salesforce API
REST API, v66.0. `GET /limits`.

Endpoint
GET /services/data/v66.0/limits

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
nearExhaustionThreshold (number, optional, default 0.8) - 0..1; flag entries above threshold.

Result
limits (object) - Map of limit name -> { Max, Remaining }. Examples:
  DailyApiRequests, DailyAsyncApexExecutions, DailyAsyncApexTests,
  DailyBulkApiBatches, DailyBulkV2QueryFileStorageMB, DailyBulkV2QueryJobs,
  DailyDeliveredPlatformEvents, DailyDurableGenericStreamingApiEvents,
  DailyDurableStreamingApiEvents, DailyEinsteinDataInsightsStoryCreation,
  DailyFunctionsApiCallLimit, DailyGenericStreamingApiEvents,
  DailyScratchOrgs, DailyStreamingApiEvents, DailyWorkflowEmails,
  DataStorageMB, FileStorageMB, HourlyAsyncReportRuns, HourlyDashboardRefreshes,
  HourlyDashboardResults, HourlyDashboardStatuses, HourlyManagedContentPublicRequests,
  HourlyOdataCallout, HourlyPublishedPlatformEvents, HourlyPublishedStandardVolumePlatformEvents,
  HourlySyncReportRuns, HourlyTimeBasedWorkflow, MassEmail, SingleEmail,
  StreamingApiConcurrentClients, etc.
nearExhaustion (array) - Limits where Used / Max ≥ threshold.
generatedAt (string).

Required permissions
- "View Setup and Configuration".

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

get_org_limits({ "nearExhaustionThreshold": 0.7 })
