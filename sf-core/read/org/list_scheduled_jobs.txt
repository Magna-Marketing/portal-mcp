Tool: list_scheduled_jobs

Description
List `CronTrigger` rows (the schedule layer of Scheduled Apex /
Scheduled Flow / Reporting Snapshots). Each shows next/previous fire
time, state (WAITING|ACQUIRED|EXECUTING|...), cron expression, owner,
and what's scheduled (CronJobDetail joined view).

Salesforce API
REST API (Data), v66.0. SOQL on `CronTrigger` and `CronJobDetail`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, OwnerId, Owner.Name, CronExpression, CronJobDetailId, CronJobDetail.Name, CronJobDetail.JobType, NextFireTime, PreviousFireTime, State, StartTime, EndTime, TimesTriggered, TimeZoneSidKey FROM CronTrigger

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
state (string, optional) - "WAITING" | "ACQUIRED" | "EXECUTING" | "DELETED" | "PAUSED" | "PAUSED_BLOCKED" | "BLOCKED" | "ERROR" | "COMPLETE".
jobType (string, optional) - "1" (Scheduled Apex) | "3" (Dashboard refresh) | "4" (ReportingSnapshot) | "8" (BatchApex) | "9" (Scheduled Flow) | "ScheduledApex" (string form).
ownerId (string, optional).
limit (number, optional, default 200).

Result
totalSize (number).
scheduledJobs (array) - { id, name, jobType, ownerId, ownerName, cronExpression, nextFireTime, previousFireTime, state, startTime, endTime, timesTriggered, timeZoneSidKey }.

Required permissions
- "View Setup and Configuration" / "Modify All Data".

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_scheduled_jobs({ "state": "WAITING" })
