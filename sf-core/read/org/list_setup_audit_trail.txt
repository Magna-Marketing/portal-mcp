Tool: list_setup_audit_trail

Description
List `SetupAuditTrail` rows: who changed what in Setup, when, with the
section and detailed action. Use for compliance reporting, change
forensics, and to confirm a deploy actually fired.

Salesforce API
REST API (Data), v66.0. SOQL on `SetupAuditTrail`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, Action, Section, Display, CreatedById, CreatedBy.Name, DelegateUser, CreatedDate, ResponsibleNamespacePrefix FROM SetupAuditTrail WHERE CreatedDate >= ... ORDER BY CreatedDate DESC

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
sinceHours (number, optional, default 24, max 4380=180 days).
section (string|string[], optional) - Filter to specific Setup sections (e.g. "Profile", "PermissionSet", "Workflow Rule", "Approval Process", "Apex Class", "Lightning Component", "Flow", "Custom Field").
userId (string, optional) - Filter to changes by this user.
namespacePrefix (string, optional) - Filter to changes from a managed package namespace.
limit (number, optional, default 500, max 50000).

Result
totalSize (number).
auditEntries (array) - { id, action, section, display, createdById, createdByName, delegateUser, createdDate, responsibleNamespacePrefix }.
hasMore (boolean).

Required permissions
- "View Setup and Configuration" / "Modify All Data".

Idempotency & limits
- Read-only.
- Retention: 180 days for most editions, 6+ months for enterprise.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_setup_audit_trail({ "section": "PermissionSet", "sinceHours": 168 })
