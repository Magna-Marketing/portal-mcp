Tool: list_identity_verification_history

Description
List `IdentityVerificationEvent` rows: MFA challenges and verification
attempts (success / failure / abandoned), method (TOTP, SMS, Email,
SecurityKey, etc.), and outcome. Use for security incident
investigation alongside `list_login_history`.

Salesforce API
REST API (Data), v66.0. SOQL on `IdentityVerificationEvent`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, UserId, EventDate, Status, Method, Activity, VerificationMethod, VerificationTime, SourceIpAddress, City, Country, Browser, Platform FROM IdentityVerificationEvent

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
userId (string, optional).
status (string, optional) - "Success" | "Failure" | "Pending" | "Skipped".
method (string, optional) - "Sms" | "Email" | "Totp" | "SecurityKey" | "U2F" | "Salesforce Authenticator" | etc.
sinceHours (number, optional, default 168).
limit (number, optional, default 500).

Result
totalSize (number).
events (array) - { id, userId, username, eventDate, status, method, activity, sourceIp, city, country, browser, platform, verificationTime }.

Required permissions
- "Manage Users" / "View Setup and Configuration".

Idempotency & limits
- Read-only.
- Retention: 180 days.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_identity_verification_history({ "status": "Failure", "sinceHours": 168 })
