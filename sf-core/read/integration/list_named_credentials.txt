Tool: list_named_credentials

Description
List Named Credentials configured in the org. Named Credentials wrap an
endpoint URL plus auth (legacy passwords, OAuth, AWS Sig V4, JWT, mTLS,
custom) and are used by Apex callouts and integrations. Each row shows
endpoint, auth provider, principal type, and managed-state.

Salesforce API
Connect API + Tooling, v66.0. `/connect/named-credentials/named-credentials`.

Endpoint
GET /services/data/v66.0/connect/named-credentials/named-credentials
GET /services/data/v66.0/connect/named-credentials/named-credentials/{developerName}

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
namePattern (string, optional).
authProvider (string, optional) - Filter by auth provider type.

Result
totalSize (number).
namedCredentials (array) - { id, developerName, masterLabel, namespacePrefix, endpoint, principalType, authProtocol, generateAuthorizationHeader, allowMergeFieldsInHeader, allowMergeFieldsInBody, calloutOptionAllowSecretInPayload, calloutSettings (object), externalCredentialId, externalCredentialName }.

Required permissions
- "Customize Application" + "Manage External Connections".

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_named_credentials({})
