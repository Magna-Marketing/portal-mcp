Tool: list_change_data_capture_channels

Description
List Change Data Capture channels and the sObjects enabled on each.
Default org channel `ChangeEvents` is shown plus any custom CDC channels.

Salesforce API
Tooling API + Metadata API, v66.0.

Endpoint
GET /services/data/v66.0/tooling/query/?q=SELECT Id, DeveloperName, MasterLabel, ChannelType FROM PlatformEventChannel WHERE ChannelType = 'data'
GET /services/data/v66.0/tooling/query/?q=SELECT EventChannelId, SelectedEntity, EnrichedFields FROM PlatformEventChannelMember WHERE EventChannel.ChannelType = 'data'
POST /services/Soap/m/66.0    (readMetadata PlatformEventSubscriberConfig for default channel)

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
includeDefaultChannel (boolean, optional, default true).
includeMembers (boolean, optional, default true).

Result
defaultChannel (object|null) - sObjects enabled on the org's `ChangeEvents` channel.
customChannels (array) - { id, developerName, masterLabel, channelType "data", members: [{ selectedEntity, enrichedFields }] }.

Required permissions
- "Customize Application".

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_change_data_capture_channels({ "includeMembers": true })
