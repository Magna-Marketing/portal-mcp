Tool: list_outbound_messages

Description
List Outbound Messages (`OutboundMessage` Metadata, the SOAP-style
webhook mechanism wired to workflow rules). Each entry shows the target
endpoint URL, fields included, and integration user used to sign the
message.

Salesforce API
Metadata API (`Workflow.outboundMessages`), v66.0.

Endpoint
POST /services/Soap/m/66.0   (readMetadata Workflow per sObject; outbound messages are inside)

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
sObjects (string[], optional).
namePattern (string, optional).
includeFailures (boolean, optional, default false) - Include `OutboundMessageDelivery` failure rows when available.

Result
totalSize (number).
outboundMessages (array) - { sObject, fullName, name, endpointUrl, integrationUserId, integrationUserName, fields (string[]), apiVersion, includeSessionId, useDeadLetterQueue, protected }.
failures (array, optional).

Required permissions
- "View Setup and Configuration" / "Customize Application".

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_outbound_messages({ "sObjects": ["Opportunity"] })
