Tool: list_external_credentials

Description
List External Credentials (modern auth provider container). Each
external credential references one auth protocol (OAuth 2.0, AWS Sig
V4, Custom, etc.) and one or more permission set mappings that grant
end-users access. Pair with `list_named_credentials` to see who calls
what.

Salesforce API
Connect API, v66.0.

Endpoint
GET /services/data/v66.0/connect/named-credentials/external-credentials
GET /services/data/v66.0/connect/named-credentials/external-credentials/{developerName}

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
namePattern (string, optional).
includePrincipalMappings (boolean, optional, default true).

Result
totalSize (number).
externalCredentials (array) - { id, developerName, masterLabel, namespacePrefix, authenticationProtocol, principals (array): [{ principalName, principalType ("PerUser"|"NamedPrincipal"|"AnonymousUser"), parameters }], permissionSetMappings (array, when includePrincipalMappings=true): [{ permissionSetName, principalName }] }.

Required permissions
- "Customize Application" + "Manage External Connections".

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_external_credentials({})
