Tool: list_attachments

Description
List legacy `Attachment` rows (now superseded by `ContentVersion`). Many
older orgs still have Attachment data. Use this to inventory and plan
migration to Files.

Salesforce API
REST API (Data), v66.0. SOQL on `Attachment`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, ParentId, Parent.Type, Name, ContentType, BodyLength, OwnerId, Owner.Name, IsPrivate, Description, CreatedDate FROM Attachment

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
parentId (string, optional).
parentSObject (string, optional) - Restrict to attachments on records of this sObject.
ownerId (string, optional).
namePattern (string, optional).
limit (number, optional, default 200).

Result
totalSize (number).
attachments (array) - { id, parentId, parentType, name, contentType, bodyLength, ownerName, isPrivate, description, createdDate }.

Required permissions
- "API Enabled" + parent record read.

Idempotency & limits
- Read-only.
- To download an attachment's body: `GET /sobjects/Attachment/{id}/Body` (separate tool: download via `dml_records` or extend later).

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_attachments({ "parentSObject": "Case" })
