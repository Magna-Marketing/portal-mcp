Tool: list_files

Description
List files in the org (`ContentDocument`) optionally scoped to a single
record (via `ContentDocumentLink`), a single owner, or a folder. Each
row carries the latest `ContentVersion` Id needed by `download_file_content`
and the parent linkages.

Salesforce API
REST API (Data) + Connect API, v66.0. SOQL on `ContentDocument`,
`ContentDocumentLink`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, Title, FileType, FileExtension, ContentSize, ContentModifiedDate, OwnerId, Owner.Name, ParentId, LatestPublishedVersionId, IsArchived FROM ContentDocument
GET /services/data/v66.0/query/?q=SELECT ContentDocumentId, LinkedEntityId, ShareType, Visibility FROM ContentDocumentLink WHERE LinkedEntityId = '...'

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
recordId (string, optional) - Restrict to files linked to this record.
ownerId (string, optional).
ownerUsername (string, optional).
fileType (string, optional) - "PDF" | "PNG" | "DOCX" | etc.
namePattern (string, optional).
includeArchived (boolean, optional, default false).
limit (number, optional, default 200).

Result
totalSize (number).
files (array) - Each: { id (ContentDocument Id), title, fileType, fileExtension, contentSize, contentModifiedDate, ownerId, ownerName, latestVersionId, isArchived, links (when recordId present): [{ linkedEntityId, shareType, visibility }] }.

Required permissions
- "API Enabled" + record visibility / file ownership.

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_files({ "recordId": "0014Q00000abcDE" })
