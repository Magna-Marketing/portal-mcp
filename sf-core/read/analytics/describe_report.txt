Tool: describe_report

Description
Get a report's metadata: type, format (tabular/summary/matrix/joined),
columns, filters, groupings, sort order, scope, and the report type
metadata (categories of available fields). Use this to understand a
report before running it (`run_report`) or before overriding filters.
Pair with `list_reports` to find candidates.

Salesforce API
Analytics REST API, v66.0.
- `GET /analytics/reports/{reportId}/describe`.

Endpoint
GET /services/data/v66.0/analytics/reports/{reportId}/describe

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
reportId (string, required) - Report Id.

Result
Verbatim Analytics REST describe payload:
attributes (object).
reportMetadata (object):
  name (string) - Report name.
  reportFormat (string) - "TABULAR" | "SUMMARY" | "MATRIX" | "JOINED".
  reportType (object) - { type (developer name), label }.
  scope (string) - "organization" | "user".
  hasDetailRows, hasRecordCount, showSubtotals, showGrandTotal (boolean).
  detailColumns (string[]) - Column API names in detail rows.
  groupingsAcross (array) - { name, sortOrder, dateGranularity }.
  groupingsDown (array) - same.
  reportFilters (array) - { column, operator, value }.
  standardDateFilter (object|null).
  reportBooleanFilter (string|null) - e.g. "(1 OR 2) AND 3".
  customDetailFormulas (array).
  aggregates (string[]).
  currency (string|null).
reportTypeMetadata (object):
  categories (array) - { name, label, columns: [{ name, label, dataType, filterable, groupable, sortable }] }.
reportExtendedMetadata (object):
  detailColumnInfo (object) - { columnApiName: { label, dataType, ... } }.
  groupingColumnInfo (object).
  aggregateColumnInfo (object).

Required permissions
- "API Enabled".
- Read on the report's folder.

Idempotency & limits
- Read-only; safe to retry.

Errors
- INVALID_REPORT_ID.
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

describe_report({ "reportId": "00OQg0000004aBcDE" })
