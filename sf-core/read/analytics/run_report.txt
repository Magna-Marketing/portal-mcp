Tool: run_report

Description
Run an existing Salesforce report and return its data. Synchronous;
caps at 2,000 rows. For larger results use `manage_report_instance`
(start an async instance and poll). Returns the raw Analytics REST
report-instance JSON: factMap, groupings, rowCounts, summary aggregates.
Use `describe_report` first if you need to know the report's columns,
filters, or grouping levels.

Salesforce API
Analytics REST API, v66.0.
- Synchronous: `GET /analytics/reports/{reportId}` (with optional query
  params for runtime overrides).
- Body POST form to override filters at runtime:
  `POST /analytics/reports/{reportId}` with reportMetadata.

Endpoint
GET  /services/data/v66.0/analytics/reports/{reportId}?includeDetails=true
POST /services/data/v66.0/analytics/reports/{reportId}    (when overriding filters)

Headers
Authorization: Bearer <access_token>
Accept: application/json
Content-Type: application/json    (only for POST)

Body
For runtime overrides (POST):
{
  "reportMetadata": {
    "reportFilters": [
      { "column": "Owner.Name", "operator": "equals", "value": "Yamin Haji" }
    ],
    "standardDateFilter": {
      "column": "CLOSE_DATE", "durationValue": "CURRENT_AND_NEXT_FQ"
    }
  }
}

Input (parameters)
reportId (string, required) - 15/18-char Report Id (begins with "00O").
includeDetails (boolean, optional, default true) - When true, include
  detail rows (factMap "T!T" key); false returns just summary aggregates.
overrideFilters (array, optional) - Runtime filter overrides; each
  { column, operator, value }. Triggers a POST.
overrideStandardDateFilter (object, optional) - { column, durationValue }.
overrideTopRows (object, optional) - { columnName, sortOrder, count }.

Result
Salesforce Analytics report-instance JSON, verbatim:
attributes (object) - { type, reportId, reportName, ... }.
reportMetadata (object) - The metadata used for this run (post-overrides).
reportTypeMetadata (object) - Categories/joinedReports info.
factMap (object) - Keyed by grouping path (e.g. "T!T", "0!T", "0_0!T"):
  aggregates (array) - Grouping aggregates with `value` and `label`.
  rows (array, when includeDetails=true) - Detail rows; each row:
    dataCells: [{ value, label }] aligned with reportMetadata.detailColumns.
groupingsAcross (object) - Column groupings.
groupingsDown (object) - Row groupings.
hasDetailRows (boolean).
allData (boolean) - false when result was truncated to 2,000 rows.

Required permissions
- "API Enabled".
- Read on the report's folder.
- Field-level read on every column the report includes.

Idempotency & limits
- Read-only; safe to retry.
- 2,000-row cap on synchronous run; async (manage_report_instance) goes
  up to 100,000 rows.
- 30-second per-call wall-clock; long reports time out.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY: caller can't see the folder.
- INVALID_REPORT_ID: bad reportId.
- REPORT_TIMEOUT: synchronous run exceeded the cap.

Example request

run_report({ "reportId": "00OQg0000004aBcDE", "includeDetails": false })
