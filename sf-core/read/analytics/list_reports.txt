Tool: list_reports

Description
List reports with optional filtering and pagination. Returns name,
folder, owner, format, last-run timestamp. The Analytics REST list
endpoint is paginated by `pageToken` (returned in the response when
there are more results).

Use `describe_report` for a single report's full metadata, `run_report`
to execute one synchronously, or `manage_report_instance` to run async.

Salesforce API
Analytics REST API, v66.0.
- `GET /analytics/reports?pageToken=...`.

Endpoint
GET /services/data/v66.0/analytics/reports?pageToken={pageToken}

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
pageToken (string, optional) - Continue from a previous page.
folderId (string, optional) - Filter to one folder Id (Folder.Id 18-char).
ownerId (string, optional) - Filter to one User Id.
namePattern (string, optional) - Glob/substring filter on report name
  (applied client-side after the API page).
limitPages (number, optional, default 1, max 50) - How many API pages
  to traverse before returning. Each page is ~200 reports.

Result
reports (array) - Each entry (raw Analytics REST shape):
  id (string) - Report Id (18-char).
  name (string).
  describeUrl (string).
  instancesUrl (string).
  folderId (string).
  folderName (string).
  ownerId (string).
  ownerName (string).
  format (string) - "Tabular" | "Summary" | "Matrix" | "Joined".
  lastRunDate (string|null).
  modifiedDate (string).
  createdDate (string).
hasNextPage (boolean).
nextPageToken (string|null).

Required permissions
- "API Enabled".

Idempotency & limits
- Read-only; safe to retry.
- API returns up to 200 reports per page.

Errors
- INVALID_PAGE_TOKEN.

Example request

list_reports({ "namePattern": "*Pipeline*", "limitPages": 3 })
