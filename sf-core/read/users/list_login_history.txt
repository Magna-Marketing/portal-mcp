Tool: list_login_history

Description
List successful and failed login attempts. Filter by user, source IP, country, application, browser, status, and date range. Use for audits ("did Jane log in from a new country yesterday?") and incident investigation. Pair with `list_identity_verification_history` for MFA challenges.

Salesforce API
REST API (Data), v66.0. SOQL on `LoginHistory`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, UserId, LoginTime, LoginType, SourceIp, LoginUrl, AuthenticationServiceId, LoginGeoId, Status, Browser, Platform, Application, ClientVersion, ApiType, ApiVersion, CountryIso, NetworkId, TlsProtocol, CipherSuite FROM LoginHistory WHERE LoginTime >= ... ORDER BY LoginTime DESC

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
userId (string, optional).
username (string, optional).
sinceHours (number, optional, default 24, max 24*180=4320 / 180 days).
status (string, optional) - "Success" | "Failed: ...".
sourceIp (string, optional) - Exact match.
country (string, optional) - ISO 2-letter.
application (string, optional) - "Browser" | "API" | "Mobile App" | etc.
loginType (string, optional) - "Application", "Application Server", "OAuth", "SAML", etc.
limit (number, optional, default 500, max 50000).

Result
totalSize (number).
logins (array) - { id, userId, username, loginTime, loginType, sourceIp, status, browser, platform, application, clientVersion, country, apiType, apiVersion, tlsProtocol }.
hasMore (boolean).

Required permissions
- "Manage Users" or "View Login Forensics" / "View Setup and Configuration".

Idempotency & limits
- Read-only.
- LoginHistory retention: 180 days (1 year on some editions).

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_login_history({ "username": "jane.doe@magnatech.com", "sinceHours": 168 })
