Tool: list_public_groups

Description
List Public Groups (Group with Type='Regular') with members, including nested groups, roles, and role+subordinates. Skip Type='Queue', Type='Role', Type='RoleAndSubordinates' synthetic rows; for those use `list_queues` or `list_user_roles`. Group members can be Users, other Groups, Roles, or Role+Subordinates — all surfaced in `members[]`.

Salesforce API
REST API (Data), v66.0. SOQL on `Group` and `GroupMember`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, Name, DeveloperName, Type, OwnerId, RelatedId, DoesIncludeBosses, DoesSendEmailToMembers FROM Group WHERE Type = 'Regular'
GET /services/data/v66.0/query/?q=SELECT Id, GroupId, UserOrGroupId FROM GroupMember WHERE GroupId IN (...)

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
namePattern (string, optional).
ownerName (string, optional) - Group OwnerId.Name match.
includeMembers (boolean, optional, default true) - Resolve UserOrGroupId into typed entries.
includeNestedExpansion (boolean, optional, default false) - Expand nested groups one level for total user reach.
limit (number, optional, default 200).

Result
totalSize (number).
groups (array) - Each:
  id, name, developerName, type "Regular", ownerId, ownerName, doesIncludeBosses, doesSendEmailToMembers.
  members (array, when includeMembers=true) - Each: { id (GroupMember Id), entityId, entityType ("User"|"Group"|"Role"|"RoleAndSubordinates"|"Queue"|"PartnerUser"|"PortalRole"|...), name }.
  effectiveUserCount (number, optional, when includeNestedExpansion=true).

Required permissions
- "View Setup and Configuration".

Idempotency & limits
- Read-only; safe to retry.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_public_groups({ "namePattern": "MTG_*", "includeMembers": true })
