Tool: list_queues

Description
List Queues with the sObjects each queue can own (`QueueSObject`) and the queue's members (users + groups). Queues are records of `Group` with Type='Queue'. Use for case routing, lead assignment, or to feed `manage_queue` updates.

Salesforce API
REST API (Data), v66.0. SOQL on `Group`, `QueueSObject`, `GroupMember`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, Name, DeveloperName, OwnerId, DoesSendEmailToMembers FROM Group WHERE Type = 'Queue'
GET /services/data/v66.0/query/?q=SELECT QueueId, SobjectType FROM QueueSObject WHERE QueueId IN (...)
GET /services/data/v66.0/query/?q=SELECT GroupId, UserOrGroupId FROM GroupMember WHERE GroupId IN (...)

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
namePattern (string, optional).
sObject (string, optional) - Restrict to queues that own this sObject.
includeMembers (boolean, optional, default true).
limit (number, optional, default 200).

Result
totalSize (number).
queues (array) - Each:
  id, name, developerName, doesSendEmailToMembers.
  supportedSObjects (string[]) - sObject API names.
  members (array, when includeMembers=true) - { entityId, entityType ("User"|"Group"|"Role"|"RoleAndSubordinates"), name }.

Required permissions
- "View Setup and Configuration".

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_queues({ "sObject": "Case", "includeMembers": true })
