Tool: list_lightning_usage

Description
Lightning Experience usage metrics: page views, exits, top users by
page, FlexiPage performance, browser/app distribution, and toggle
metrics. Useful for adoption reporting and performance tuning.

Salesforce API
REST API, v66.0. `/lightning/usage*`.

Endpoint
GET /services/data/v66.0/lightning/usagetoggle?startDate=...&endDate=...                    (toggles)
GET /services/data/v66.0/lightning/usagebypage?startDate=...&endDate=...                    (per page)
GET /services/data/v66.0/lightning/usagebybrowser?startDate=...&endDate=...                 (browsers)
GET /services/data/v66.0/lightning/usagebyapptype?startDate=...&endDate=...                 (app types)
GET /services/data/v66.0/lightning/usagebyflexipage?startDate=...&endDate=...               (FlexiPages)
GET /services/data/v66.0/lightning/exitbypage?startDate=...&endDate=...                     (exit metrics)

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
slice (string, required) - "toggle" | "byPage" | "byBrowser" | "byAppType" | "byFlexiPage" | "exitByPage".
startDate (string, required) - YYYY-MM-DD.
endDate (string, required) - YYYY-MM-DD.

Result
slice (string).
metrics (array) - Shape varies per slice. Common keys: { date, pageName, browser, appType, viewCount, uniqueUserCount, exitCount }.

Required permissions
- "View Setup and Configuration" + "Modify All Data" (for some slices).

Idempotency & limits
- Read-only.
- Date range max 30 days per call.

Errors
- INVALID_DATE_RANGE, INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_lightning_usage({ "slice": "byFlexiPage", "startDate": "2026-04-01", "endDate": "2026-04-29" })
