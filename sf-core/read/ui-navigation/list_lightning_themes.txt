Tool: list_lightning_themes

Description
List Lightning Experience Themes (`LightningExperienceTheme`) and the
active theme. Themes control header colors, logo, brand, accent, etc.

Salesforce API
Metadata API (`LightningExperienceTheme`), v66.0.

Endpoint
POST /services/Soap/m/66.0   (readMetadata LightningExperienceTheme)
GET /services/data/v66.0/tooling/query/?q=SELECT Id, DeveloperName, MasterLabel FROM LightningExperienceTheme

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
includeBranding (boolean, optional, default true) - Include color hex codes and image references.

Result
activeTheme (string|null) - DeveloperName of the active theme.
themes (array) - Each:
  id, developerName, masterLabel, description, defaultBrandingSet (string|null).
  branding (object, when includeBranding=true) - { headerColor, primaryColor, secondaryColor, accentColor, brandImageUrl, logoImageUrl, faviconUrl, backgroundImageUrl }.

Required permissions
- "View Setup and Configuration" / "Customize Application".

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_lightning_themes({ "includeBranding": true })
