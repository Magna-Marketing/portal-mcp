Tool: view_permission_set

Description
Get the full detail of a permission set: identity, license, system
permissions (the boolean Permissions* flags), object permissions
(CRUD + viewAll/modifyAll per sObject), field permissions (read/edit
per field), Apex class access, Visualforce page access, custom tab
visibility, and user-license / permission-set-license restrictions.
Use this before assigning a permset (`assign_permission_set`) or
auditing what a group of users actually gets. Pair with
`list_permission_set_assignments` to find who's already on it.

Salesforce API
REST API + Tooling API, v66.0.
- Identity & system perms: SOQL on `PermissionSet`.
- Object perms: SOQL on `ObjectPermissions WHERE ParentId = '<permsetId>'`.
- Field perms: SOQL on `FieldPermissions WHERE ParentId = '<permsetId>'`.
- Apex/VF access: `SetupEntityAccess` filtered by SetupEntityType.
- License: SOQL on `PermissionSet.LicenseId` and `PermissionSetLicense`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, Name, Label, Description, Type, Profile.Name, License.MasterLabel, IsCustom, IsOwnedByProfile, NamespacePrefix, Permissions* FROM PermissionSet WHERE Name = '{name}' OR Id = '{id}'
GET /services/data/v66.0/query/?q=SELECT SObjectType, PermissionsRead, PermissionsCreate, PermissionsEdit, PermissionsDelete, PermissionsViewAllRecords, PermissionsModifyAllRecords FROM ObjectPermissions WHERE ParentId = '{permsetId}'
GET /services/data/v66.0/query/?q=SELECT SObjectType, Field, PermissionsRead, PermissionsEdit FROM FieldPermissions WHERE ParentId = '{permsetId}'
GET /services/data/v66.0/tooling/query/?q=SELECT SetupEntityType, SetupEntityId FROM SetupEntityAccess WHERE ParentId = '{permsetId}'

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None (GET requests).

Input (parameters)
name (string, optional) - Permission set Name (DeveloperName-equivalent).
id (string, optional) - 15/18-char permission set Id.
  One of `name` or `id` is required.
includeFieldPermissions (boolean, optional, default true) - When false,
  omit the (often large) field permissions array.
includeApexAccess (boolean, optional, default true).
includeVfAccess (boolean, optional, default true).
includeTabSettings (boolean, optional, default true).
includeUserCount (boolean, optional, default true) - Add the current
  number of assigned users (one extra COUNT query).

Result
permissionSet (object):
  id, name, label, description, type ("Regular"|"Session"|"Group"),
  isCustom (boolean), isOwnedByProfile (boolean),
  namespacePrefix (string|null),
  license (object|null) - { id, name, totalLicenses, usedLicenses }.
  systemPermissions (object) - Map of permission name -> boolean (e.g.
    "ApiEnabled": true, "ManageUsers": false, "ModifyAllData": false).
    Only `true` entries are returned by default; pass `includeFalse=true`
    to see explicit false flags.
  objectPermissions (array) - Each: { sObject, read, create, edit,
    delete, viewAll, modifyAll }.
  fieldPermissions (array, optional) - Each: { sObject, field, read, edit }.
  apexClassAccess (array) - { id, name, namespacePrefix }.
  vfPageAccess (array) - { id, name }.
  tabSettings (array) - { tabName, visibility ("Available"|"DefaultOn"|"DefaultOff"|"Hidden") }.
  userCount (number, optional).

Required permissions
- "View Setup and Configuration" + read on `PermissionSet`,
  `ObjectPermissions`, `FieldPermissions`, `SetupEntityAccess`. (Granted
  by default to system administrators.)

Idempotency & limits
- Read-only; safe to retry.
- For permsets with many fields the FieldPermissions query can return
  >10k rows; the tool paginates internally.

Errors
- NOT_FOUND: permission set name/id not present.
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

view_permission_set({ "name": "MTG_Project_Manager", "includeFieldPermissions": false })
