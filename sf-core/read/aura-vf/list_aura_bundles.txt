Tool: list_aura_bundles

Description
List Aura component bundles (`AuraDefinitionBundle`) with API version,
description, and the count of definitions inside (Component, Application, Event, Interface, Style, Documentation, etc.).

Salesforce API
Tooling API, v66.0.

Endpoint
GET /services/data/v66.0/tooling/query/?q=SELECT Id, DeveloperName, MasterLabel, ApiVersion, NamespacePrefix, Description, ManageableState, CreatedDate, LastModifiedDate FROM AuraDefinitionBundle
GET /services/data/v66.0/tooling/query/?q=SELECT Id, AuraDefinitionBundleId, DefType FROM AuraDefinition WHERE AuraDefinitionBundleId IN (...)

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
namePattern (string, optional).
namespacePrefix (string, optional).
includeManaged (boolean, optional, default false).
includeDefinitionCounts (boolean, optional, default true).
limit (number, optional, default 200).

Result
totalSize (number).
bundles (array) - { id, developerName, masterLabel, apiVersion, namespacePrefix, description, manageableState, lastModifiedDate, definitionCounts (object): { COMPONENT, APPLICATION, EVENT, INTERFACE, STYLE, CONTROLLER, HELPER, RENDERER, DOCUMENTATION, DESIGN, SVG } }.

Required permissions
- "Customize Application".

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_aura_bundles({ "namePattern": "*Card" })
