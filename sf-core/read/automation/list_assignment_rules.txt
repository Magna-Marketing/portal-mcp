Tool: list_assignment_rules

Description
List Lead and Case assignment rules: rule entries (criteria, target user/queue, email template, notify checkbox), and active rule per object. There can be many rules per object but only one is active at a time per Salesforce constraint.

Salesforce API
Metadata API (`AssignmentRules` per sObject), v66.0.

Endpoint
POST /services/Soap/m/66.0   (readMetadata AssignmentRules)

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
sObjects (string[], optional, default ["Lead", "Case"]).
includeInactive (boolean, optional, default true).

Result
assignmentRules (array) - Each:
  sObject (string).
  fullName (string) - Rule name.
  active (boolean).
  ruleEntries (array) - Each:
    sortOrder (number).
    criteriaItems (array) - [{ field, operation, value }].
    formula (string|null).
    booleanFilter (string|null).
    assignedTo (string).
    assignedToType ("User"|"Queue"|"PartnerUser"|"Owner").
    emailTemplate (string|null).
    notifyCcRecipients (boolean).
    overrideExistingTeams (boolean, only Case).

Required permissions
- "View Setup and Configuration" / "Customize Application".

Idempotency & limits
- Read-only.

Errors
- INVALID_TYPE on bad sObject (only Lead and Case support assignment rules).

Example request

list_assignment_rules({ "sObjects": ["Case"] })
