Tool: list_escalation_rules

Description
List Case escalation rules and their entries (criteria, business hours, escalation actions). Escalation actions can reassign the case, notify users, and update fields based on age/status.

Salesforce API
Metadata API (`EscalationRules`), v66.0.

Endpoint
POST /services/Soap/m/66.0   (readMetadata EscalationRules for Case)

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
includeInactive (boolean, optional, default true).
namePattern (string, optional).

Result
escalationRules (array) - Each:
  fullName, active.
  ruleEntries (array) - Each:
    sortOrder, criteriaItems[], formula, booleanFilter,
    businessHours (string|null) - BusinessHours name,
    escalationActions (array) - [{ ageOver (minutes), ageOverFromField, assignTo (string), assignToTemplate (string), notifyTo (string|null), notifyTemplate (string|null), notifyAssignee (boolean) }].

Required permissions
- "View Setup and Configuration" / "Customize Application".

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_escalation_rules({ "includeInactive": false })
