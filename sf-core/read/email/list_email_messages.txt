Tool: list_email_messages

Description
List `EmailMessage` rows logged against records (Cases, Leads, Contacts,
custom). Use to audit Email-to-Case threads or programmatic email
sending. Each row includes from/to/cc/bcc, subject, body (when
requested), parent record, status, attachments count.

Salesforce API
REST API (Data), v66.0. SOQL on `EmailMessage`.

Endpoint
GET /services/data/v66.0/query/?q=SELECT Id, ParentId, Parent.Name, FromAddress, FromName, ToAddress, CcAddress, BccAddress, Subject, Status, Incoming, MessageDate, HasAttachment, ThreadIdentifier FROM EmailMessage WHERE ...

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
parentId (string, optional) - Restrict to one record (Case Id, etc.).
fromAddress (string, optional).
toAddress (string, optional).
sinceHours (number, optional).
includeBody (boolean, optional, default false) - Include `TextBody` and `HtmlBody`.
incoming (boolean, optional) - Filter inbound vs outbound.
limit (number, optional, default 200).

Result
totalSize (number).
emails (array) - { id, parentId, parentName, fromAddress, fromName, toAddress, ccAddress, bccAddress, subject, status, incoming (boolean), messageDate, hasAttachment, threadIdentifier, textBody (optional), htmlBody (optional) }.

Required permissions
- "API Enabled" + read on the parent sObject.

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_email_messages({ "parentId": "500Qg0000004ZyAA", "includeBody": true })
