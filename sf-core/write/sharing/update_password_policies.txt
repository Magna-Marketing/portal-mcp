Tool: update_password_policies

Description
Update org-level password policies (length, complexity, expiration,
lockout interval, lockout attempts) and per-profile overrides.

Salesforce API
Metadata API (`SecuritySettings`, `ProfilePasswordPolicy`), v66.0.

Endpoint
POST /services/data/v66.0/metadata/deployRequest

Headers
Authorization: Bearer <access_token>
Content-Type: application/json

Body
N/A.

Input (parameters)
password (object, optional) - { expiration, minimumLength, complexity, historyRestriction, maxLoginAttempts, lockoutInterval, questionRestriction, obscureSecretAnswer }.
profileOverrides (object[], optional) - [{ profileName, expiration, minimumLength, complexity, historyRestriction, maxLoginAttempts, lockoutInterval }].
checkOnly (boolean, optional, default false).

Result
deploy (object).
password (object).
profileOverrides (array).

Required permissions
- "Manage Internal Users".

Idempotency & limits
- Idempotent.

Errors
- INVALID_FIELD on bad enum.

Example request

update_password_policies({
  "password": { "minimumLength": 12, "complexity": "UpperLowerCaseNumberSpecialCharacters", "expiration": "NinetyDays", "historyRestriction": 5, "maxLoginAttempts": 5, "lockoutInterval": "ThirtyMinutes" }
})
