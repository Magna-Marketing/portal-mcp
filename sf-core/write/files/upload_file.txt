Tool: upload_file

Description
Upload a file to Salesforce Files. Creates a `ContentVersion` (which
auto-creates a `ContentDocument`). Optionally link to one or more
records (`ContentDocumentLink`) and set sharing visibility.

Salesforce API
REST API (Data), v66.0. `POST /sobjects/ContentVersion` with
multipart/form-data, then `POST /sobjects/ContentDocumentLink`.

Endpoint
POST /services/data/v66.0/sobjects/ContentVersion           (multipart)
POST /services/data/v66.0/sobjects/ContentDocumentLink      (per link)

Headers
Authorization: Bearer <access_token>
Content-Type: multipart/form-data; boundary=...        (set by tool)

Body
Multipart with two parts: JSON metadata (`PathOnClient`, `Title`,
`Description`, `FirstPublishLocationId`) and binary `VersionData`.

Input (parameters)
fileName (string, required) - Display name + extension.
title (string, optional, default = fileName).
description (string, optional).
base64 (string, required) - Base64-encoded contents.
linkedToRecordIds (string[], optional) - Record Ids to attach to.
shareType (string, optional, default "V") - "V" (viewer) | "C" (collaborator) | "I" (inferred).
visibility (string, optional, default "AllUsers") - "AllUsers" | "InternalUsers" | "SharedUsers".
firstPublishLocationId (string, optional) - Where the file is "owned" (record Id, library Id, or own user Id).

Result
file (object) - { contentDocumentId, contentVersionId, title, fileType, fileExtension, contentSize }.
links (array) - Created CDL rows: { id, linkedEntityId, shareType, visibility }.

Required permissions
- "API Enabled" + write access on the target records.

Idempotency & limits
- Not idempotent.
- Default per-file size: 2 GB (org-configurable).

Errors
- FILE_TOO_LARGE, INSUFFICIENT_ACCESS_OR_READONLY.

Example request

upload_file({
  "fileName": "MTG_Project_Brief.pdf",
  "base64": "JVBERi0xLjQK...",
  "linkedToRecordIds": ["a01Qg0000020aBcDE"]
})
