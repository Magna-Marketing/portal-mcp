Tool: process_approval_action

Description
Approve, reject, or recall an in-flight approval. Acts on a
`ProcessInstanceWorkitem` by Id (visible to the running user as the
approver).

Salesforce API
REST API (Process Approvals), v66.0. Same `POST /process/approvals` endpoint with `actionType` "Approve" | "Reject" | "Removed".

Endpoint
POST /services/data/v66.0/process/approvals/

Headers
Authorization: Bearer <access_token>
Content-Type: application/json

Body
{
  "requests": [{
    "actionType": "Approve",
    "contextId": "<workitem id>",
    "comments": "LGTM",
    "nextApproverIds": []
  }]
}

Input (parameters)
records (object[], required) - Each:
  workitemId (string, optional) - ProcessInstanceWorkitem Id.
  recordId (string, optional) - Record under approval; tool resolves to current workitem for running user.
  action (string, required) - "approve" | "reject" | "recall".
  comments (string, optional).
  nextApproverIds (string[], optional) - For multi-step approvals where the next step needs explicit approver.

Result
results (array) - { workitemId, instanceId, action, instanceStatus, errors }.

Required permissions
- Be a permitted actor on the workitem (or have "Modify All Data" for recall).

Idempotency & limits
- Not idempotent — once approved/rejected, the workitem is gone.

Errors
- INVALID_OPERATION_WITH_EXPIRED_APEX_REST_RESOURCE, INSUFFICIENT_ACCESS_OR_READONLY.

Example request

process_approval_action({
  "records": [{ "workitemId": "04iQg00000abcDE", "action": "approve", "comments": "Approved" }]
})
