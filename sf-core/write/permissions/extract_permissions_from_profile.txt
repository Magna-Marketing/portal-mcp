Tool: extract_permissions_from_profile

Description
Extract a chosen subset of permissions from an existing profile into a
brand-new permission set. Useful for incremental profile-to-permset
migrations: pick the slice that's reusable, lift it out, drop it from
the profile (optional), then assign the new permset to users.

Common subsets:
- Object permissions (CRUD on a specific object).
- Field permissions for a category of fields.
- API perms (ApiEnabled, ManageDataIntegrations, ...).
- Custom object access (a single __c object).

Salesforce API
Metadata API (`Profile` readMetadata + `PermissionSet` create), v66.0.

Endpoint
POST /services/Soap/m/66.0    (readMetadata Profile)
POST /services/data/v66.0/metadata/deployRequest

Headers
Authorization: Bearer <access_token>
Content-Type: application/json

Body
The MCP server reads the source profile, copies the requested
sub-permissions, and submits a `create` for the new permset.

Input (parameters)
sourceProfile (string, required) - Profile DeveloperName.
newPermsetName (string, required) - DeveloperName of the new permset.
newPermsetLabel (string, required).
newPermsetDescription (string, optional).
license (string, optional) - Permset license; default = source profile's
  user license.
extract (object, required) - What to extract:
  systemPermissions (string[], optional) - User permission API names,
    e.g. ["ApiEnabled", "RunReports"].
  objectPermissions (object[], optional) - { object, accessTypes:
    ["read","create","edit","delete","viewAll","modifyAll"] }.
  fieldPermissions (object[], optional) - { object, fields: ["Name","Phone"] }.
  apexClasses (string[], optional).
  vfPages (string[], optional).
  customMetadataTypes (string[], optional).
  customSettings (string[], optional).
  tabs (object[], optional) - { tab, visibility }.
removeFromProfile (boolean, optional, default false) - When true, the
  same permissions are removed from the source profile in a follow-up
  Metadata deploy. Use with care.
checkOnly (boolean, optional, default false).

Result
permissionSet (object) - { id, fullName, label, license, copiedComponents }.
profileUpdate (object|null) - When removeFromProfile=true: {
  deployId, status, removedComponents, succeeded }.

Required permissions
- "Manage Profiles and Permission Sets".
- "Modify All Data" if removeFromProfile=true.

Idempotency & limits
- New permset creation: not idempotent (fails on duplicate).
- Remove-from-profile: idempotent (no-op when permissions are already
  absent).

Errors
- DUPLICATE_DEVELOPER_NAME on newPermsetName.
- INVALID_FIELD on bad field reference.

Example request

extract_permissions_from_profile({
  "sourceProfile": "MTG_Project_Manager_Profile",
  "newPermsetName": "MTG_Project_API_Access",
  "newPermsetLabel": "MTG Project | API Access",
  "extract": {
    "systemPermissions": ["ApiEnabled", "ManageDataIntegrations"],
    "objectPermissions": [{ "object": "Project__c", "accessTypes": ["read", "edit"] }]
  },
  "removeFromProfile": false
})
