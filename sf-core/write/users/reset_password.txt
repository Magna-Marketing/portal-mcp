Tool: reset_password

Description
Reset a user's password. Two modes:
1. **Send reset email**: triggers Salesforce to email the user a reset
   link. No password is exposed.
2. **Generate temporary password**: returns a one-time password the
   admin can share (the user is forced to change on next login).

Salesforce API
REST API (Data), v66.0. `/sobjects/User/{id}/password` (POST = generate temp password; DELETE = expire current password).

Endpoint
POST  /services/data/v66.0/sobjects/User/{userId}/password    (sets a new password)
DELETE /services/data/v66.0/sobjects/User/{userId}/password   (expires; user must reset)
POST  /services/data/v66.0/sobjects/User/{userId}/passwordResetEmail   (REST API in Connect)

Headers
Authorization: Bearer <access_token>
Content-Type: application/json

Body
For POST password: { "NewPassword": "Temp!1234" }   (or omit to let SF generate)

Input (parameters)
userId (string, optional).
username (string, optional).
  One required.
mode (string, required) - "sendResetEmail" | "generateTempPassword" | "expirePassword".
newPassword (string, optional) - Used with `generateTempPassword`. Must satisfy org's password policy.

Result
user (object) - { id, username }.
mode (string).
generatedPassword (string|null) - Returned only when mode="generateTempPassword" and `newPassword` was not provided.
emailSent (boolean) - true when mode="sendResetEmail".

Required permissions
- "Reset User Passwords and Unlock Users" / "Manage Internal Users".

Idempotency & limits
- Not idempotent (each call invalidates any prior session).

Errors
- NOT_FOUND, INSUFFICIENT_ACCESS_OR_READONLY, PASSWORD_LOCKOUT (user already locked).

Example request (email reset link)

reset_password({ "username": "jane.doe@magnatech.com", "mode": "sendResetEmail" })

Example request (generate temp password)

reset_password({ "username": "jane.doe@magnatech.com", "mode": "generateTempPassword" })
