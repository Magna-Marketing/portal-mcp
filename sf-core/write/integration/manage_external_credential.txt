Tool: manage_external_credential

Description
Create / update / delete External Credentials and their principals
(NamedPrincipal / PerUser / AnonymousUser). Map permission sets to
principals so callouts use the right credentials per user.

Salesforce API
Connect API, v66.0.

Endpoint
POST   /services/data/v66.0/connect/named-credentials/external-credentials
PATCH  /services/data/v66.0/connect/named-credentials/external-credentials/{developerName}
DELETE /services/data/v66.0/connect/named-credentials/external-credentials/{developerName}

Headers
Authorization: Bearer <access_token>
Content-Type: application/json

Body
N/A.

Input (parameters)
operation (string, required) - "create" | "update" | "delete" | "addPrincipal" | "removePrincipal" | "mapPermissionSet" | "unmapPermissionSet".
developerName (string, required).
masterLabel (string, required for create).
authenticationProtocol (string, required for create) - "OAuth20" | "AwsSv4" | "Custom" | "NoAuthentication".
principals (object[], optional) - Each: { principalName, principalType ("PerUser"|"NamedPrincipal"|"AnonymousUser"), parameters (object) - protocol-specific (e.g. tokenEndpointUrl, clientId, scope) }.
permissionSetMappings (object[], optional) - Each: { permissionSetName, principalName }.

Result
externalCredential (object) - { id, developerName, masterLabel, authenticationProtocol, principals, permissionSetMappings }.

Required permissions
- "Customize Application" + "Manage External Connections".

Idempotency & limits
- update / addPrincipal / mapPermissionSet idempotent.

Errors
- DUPLICATE_DEVELOPER_NAME, INVALID_FIELD.

Example request

manage_external_credential({
  "operation": "create",
  "developerName": "MTG_Partner_OAuth",
  "masterLabel": "MTG Partner OAuth",
  "authenticationProtocol": "OAuth20",
  "principals": [{ "principalName": "default", "principalType": "PerUser", "parameters": { "scope": "read write" } }],
  "permissionSetMappings": [{ "permissionSetName": "MTG_Partner_API_User", "principalName": "default" }]
})
