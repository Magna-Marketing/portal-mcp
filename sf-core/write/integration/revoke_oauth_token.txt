Tool: revoke_oauth_token

Description
Revoke OAuth tokens for a user, an app, or a specific token Id.
Forces re-authentication on next call.

Salesforce API
REST API + Connect API, v66.0.

Endpoint
DELETE /services/data/v66.0/sobjects/OauthToken/{id}                 (single token)
DELETE /services/data/v66.0/apps/oauth/usage/{appId}/users/{userId}/tokens     (all tokens for user/app)

Headers
Authorization: Bearer <access_token>

Body
None.

Input (parameters)
mode (string, required) - "byTokenId" | "byUserAndApp" | "byUser" | "byApp".
tokenId (string, optional, required for "byTokenId").
userId (string, optional) / username (string, optional).
appId (string, optional) / appName (string, optional).

Result
revoked (number).
results (array) - { tokenId, status ("revoked"|"failed"), error }.

Required permissions
- "Manage Connected Apps" + "Manage Users".

Idempotency & limits
- Idempotent.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY, NOT_FOUND.

Example request (revoke all tokens for a user across all apps)

revoke_oauth_token({ "mode": "byUser", "username": "compromised@magnatech.com" })
