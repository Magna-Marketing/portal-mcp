Tool: trigger_identity_resolution

Description
Run an Identity Resolution ruleset — re-evaluates the match +
reconciliation rules against the current source DMOs to refresh
the Unified Individual / Unified Account profiles. Async; returns
a `runId` and the run executes in the background. Subsequent
profile reads (`get_unified_profile`) and segment publishes use
the refreshed unified data.

Salesforce API
Data Cloud Identity Resolution API.

Endpoint
POST {cdpInstanceUrl}/api/v1/identityresolution/rulesets/:rulesetId/actions/run

Headers
Authorization: Bearer <data_cloud_token>
Content-Type: application/json
Accept: application/json

Body
{
  "runMode": "Incremental"
}

Input (parameters)
rulesetId (string, required).
dataSpace (string, optional, default "default").
runMode (string, optional, default "Incremental") - "Full" |
  "Incremental".
runIdempotencyKey (string, optional).

Result
runId (string).
rulesetId (string).
status (string) - "Queued".
submittedAt (ISO-8601).

Required scopes
- `cdp_identityresolution_api`.

Idempotency & limits
- Idempotent on `runIdempotencyKey`.
- 1 in-flight run per ruleset.
- A `Full` run on a 10M+ record graph can take 30-90 minutes;
  pair with `list_identity_resolution_rulesets` polling.

Errors
- 404 NOT_FOUND, 409 JOB_IN_PROGRESS,
  503 SOURCE_DMO_UNAVAILABLE (a source DMO is paused / failed),
  401 invalid_token.

Example request

trigger_identity_resolution({
  "rulesetId": "IR_UNIF_IND_DEFAULT",
  "runMode": "Incremental"
})

Example result
{
  "runId": "RUN_IR_2026-04-29T13",
  "rulesetId": "IR_UNIF_IND_DEFAULT",
  "status": "Queued",
  "submittedAt": "2026-04-29T13:02:11Z"
}
