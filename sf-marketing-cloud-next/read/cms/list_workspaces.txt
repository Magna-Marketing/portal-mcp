Tool: list_workspaces

Description
List CMS Workspaces (Content Spaces) in the org. A Workspace is a
top-level container for marketing content; emails, templates, and
assets all live in exactly one workspace. Either list all
workspaces or look one up by `name`.

Salesforce API
Connect CMS API.

Endpoint
GET {instanceUrl}/services/data/v{ver}/connect/cms/spaces?name={name}

Headers
Authorization: Bearer <core_access_token>
Accept: application/json

Body
None.

Input (parameters)
name (string, optional) - Workspace developer name; when supplied,
  the response is a single workspace under `spaces[0]`.
includeFolders (boolean, optional, default false) - When true, also
  inlines the top-level folder list per workspace.

Result
spaces (array) - Each: { id, name, label, description, language,
  isDefault, createdDate, lastModifiedDate, channels (array of channel ids), folders (array, when includeFolders=true) }.
nextPageToken (string|null).

Required scopes
- `api`, `cms_api`.

Idempotency & limits
- Read-only.

Errors
- 401 INVALID_SESSION_ID, 403 INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_workspaces({})

Example result
{
  "spaces": [
    { "id": "0Z0xx0000000001AAA", "name": "marketing_main",
      "label": "Marketing Main", "isDefault": true,
      "channels": ["MarketingCloudNext_Email"] }
  ]
}
