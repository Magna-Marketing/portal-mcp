Tool: list_journeys

Description
List Journey Builder journeys (`/interaction/v1/interactions`). Each
journey row includes Id, name, status (Draft / Published / Stopped /
Paused), version, last published date, and entry mode.

Marketing Cloud API
REST API (`/interaction/v1/interactions`).

Endpoint
GET https://{tenant}.rest.marketingcloudapis.com/interaction/v1/interactions?$page=1&$pageSize=50

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
status (string|string[], optional) - "Draft" | "Published" | "ScheduledToPublish" | "Stopped" | "Paused" | "Unpublished" | "Deleted".
namePattern (string, optional).
modifiedSince (string, optional).
mostRecentVersionOnly (boolean, optional, default true).
includeHistory (boolean, optional, default false).
page (number, optional, default 1).
pageSize (number, optional, default 50, max 200).

Result
count (number).
items (array) - { id, key, name, description, version, status, definitionType ("Multistep"|"Quicksend"|"Transactional"), workflowApiVersion, lastPublishedDate, createdDate, modifiedDate, entryMode ("MultipleEntries"|"NotReEntrant"|"OncePerContact"|"NoReEntry") }.
hasMore (boolean), page (number).

Required scopes
- `journeys_read`.

Idempotency & limits
- Read-only.

Errors
- 10000, 401 unauthorized.

Example request

list_journeys({ "status": "Published" })
