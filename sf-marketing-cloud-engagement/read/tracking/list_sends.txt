Tool: list_sends

Description
List recent send-tracking events (`SentEvent`). One row per
subscriber-per-send: when the platform actually queued/sent the email
to the SubscriberKey.

Marketing Cloud API
SOAP API (`SentEvent.Retrieve`).

Endpoint
POST https://{tenant}.soap.marketingcloudapis.com/Service.asmx

Headers
Authorization: Bearer <access_token>
Content-Type: text/xml; charset=utf-8

Body
SOAP retrieve on SentEvent with EventDate filter.

Input (parameters)
sinceDays (number, optional, default 1).
sendId (number, optional).
subscriberKey (string, optional).
batchId (number, optional).
limit (number, optional, default 1000).

Result
totalSize (number).
sends (array) - { sendId, subscriberKey, subscriberId, listId, batchId, eventDate, triggererSendDefinitionObjectId (string|null) }.
hasMore (boolean), continueRequestId (string|null).

Required scopes
- `tracking_events_read`.

Idempotency & limits
- Read-only.
- Tracking event volume can be large; prefer narrow time windows.

Errors
- 401 unauthorized.

Example request

list_sends({ "sinceDays": 1 })
