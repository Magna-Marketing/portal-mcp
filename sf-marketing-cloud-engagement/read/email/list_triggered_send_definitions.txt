Tool: list_triggered_send_definitions

Description
List `TriggeredSendDefinition` records — the configurations behind
triggered sends (single emails fired from APIs / Journey Builder).
Each definition has its email asset, send classification, status (Active
/ Inactive / Paused), and per-call options.

Marketing Cloud API
SOAP API (`TriggeredSendDefinition.Retrieve`).

Endpoint
POST https://{tenant}.soap.marketingcloudapis.com/Service.asmx

Headers
Authorization: Bearer <access_token>
Content-Type: text/xml; charset=utf-8

Body
SOAP retrieve on TriggeredSendDefinition.

Input (parameters)
status (string|string[], optional) - "New" | "Active" | "Inactive" | "Paused" | "Cancelled" | "Deleted".
namePattern (string, optional).
folderId (number, optional).
emailCustomerKey (string, optional) - Filter to one email asset.
limit (number, optional, default 200).

Result
totalSize (number).
triggeredSends (array) - { id, customerKey, name, description, triggeredSendStatus, emailId (Email Studio email ref), emailContentCheck, sendClassificationCustomerKey, senderProfileCustomerKey, deliveryProfileCustomerKey, fromName, fromAddress, isMultipart, replyToAddress, replyToDisplayName, autoAddSubscribers, autoUpdateSubscribers, batchInterval, refreshContent, suppressTracking, isSendLogging, sendSourceCustomerKey, dateCreated, dateModified }.

Required scopes
- `email_read`.

Idempotency & limits
- Read-only.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY.

Example request

list_triggered_send_definitions({ "status": "Active" })
