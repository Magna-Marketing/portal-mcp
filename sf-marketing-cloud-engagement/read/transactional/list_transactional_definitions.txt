Tool: list_transactional_definitions

Description
List Transactional Messaging definitions (email + SMS). Each
definition is a stable shape (subject, content reference, sender, send
classification) addressed by a `definitionKey`; `send_transactional_email`
/ `send_transactional_sms` invoke a definition to actually send.

Marketing Cloud API
REST API (`/messaging/v1/email/definitions`, `/messaging/v1/sms/definitions`).

Endpoint
GET https://{tenant}.rest.marketingcloudapis.com/messaging/v1/email/definitions
GET https://{tenant}.rest.marketingcloudapis.com/messaging/v1/sms/definitions

Headers
Authorization: Bearer <access_token>
Accept: application/json

Body
None.

Input (parameters)
type (string, required) - "email" | "sms".
status (string|string[], optional) - "Active" | "Inactive" | "Deleted".
namePattern (string, optional).
page (number, optional, default 1).
pageSize (number, optional, default 50, max 50).

Result
count (number).
definitions (array) - { definitionKey, definitionId, name, description, status, classification, content (object) - email/sms specific, subscriptions (object), createdDate, modifiedDate }.

Required scopes
- `email_send` (email) / `sms_send` (sms).

Idempotency & limits
- Read-only.

Errors
- 10000 bad request.

Example request

list_transactional_definitions({ "type": "email", "status": "Active" })
