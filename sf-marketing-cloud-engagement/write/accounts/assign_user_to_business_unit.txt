Tool: assign_user_to_business_unit

Description
Grant / revoke a user's access to a business unit (MID). Each user can
be allowed into multiple BUs in an Enterprise hierarchy.

Marketing Cloud API
SOAP API (`AccountUser` Update with `AssociatedBusinessUnits__c`).

Endpoint
POST https://{tenant}.soap.marketingcloudapis.com/Service.asmx

Headers
Authorization: Bearer <access_token>
Content-Type: text/xml; charset=utf-8

Body
SOAP UpdateRequest.

Input (parameters)
userId (number, optional).
username (string, optional).
  One required.
operation (string, required) - "add" | "remove" | "setDefault".
businessUnitIds (number[], optional) - MIDs to add / remove.
defaultBusinessUnitId (number, optional, only for setDefault).

Result
user (object) - { id, username, defaultBusinessUnitId, businessUnits: [{ id, name, isDefault }] }.
changes (object).

Required scopes
- `users_write` + Enterprise edition.

Idempotency & limits
- add / remove idempotent.

Errors
- INSUFFICIENT_ACCESS_OR_READONLY (non-Enterprise).

Example request

assign_user_to_business_unit({
  "username": "marketingops@magnatech.com",
  "operation": "add",
  "businessUnitIds": [5001234, 5001235]
})
