Tool: get_list_email

Description
Read a single List Email by id. Returns the same shape as
`list_list_emails` plus the rendered `htmlMessage`/`textMessage` and
the resolved list-of-lists targeting. Use after a send completes
to pull final aggregate stats.

Salesforce API
Account Engagement v5 REST API.

Endpoint
GET https://{pi-domain}/api/v5/objects/list-emails/:id?fields={fields}

Headers
Authorization: Bearer <salesforce-access-token>
Pardot-Business-Unit-Id: <buId>
Accept: application/json

Body
None.

Input (parameters)
id (number, required).
fields (string[], optional) - Default expands `htmlMessage`,
  `textMessage`, `lists.*`, `suppressionLists.*`,
  `template.*`.

Result
listEmail (object): id, name, subject, htmlMessage, textMessage,
  status, sentAt, scheduledTime, campaignId, listIds (array),
  suppressionListIds (array), emailTemplateId, operationalEmail,
  senderOptions, replyToOptions, trackerDomainId, sentCount,
  deliveredCount, uniqueOpenCount, totalOpenCount, uniqueClickCount,
  totalClickCount, bounceCount, unsubscribeCount, spamComplaintCount,
  optOutCount, deliveryRate, openRate, clickRate, createdAt, updatedAt.

Required scopes
- `pardot_api`.

Idempotency & limits
- Read-only.

Errors
- 59 not_found.

Example request

get_list_email({ "id": 8821 })

Example result
{
  "listEmail": {
    "id": 8821, "name": "Spring Promo Send", "status": "Sent",
    "sentAt": "2026-04-21T15:00:00Z",
    "sentCount": 12450, "openRate": 36.7, "clickRate": 8.4,
    "listIds": [301, 302]
  }
}
