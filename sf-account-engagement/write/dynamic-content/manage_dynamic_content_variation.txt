Tool: manage_dynamic_content_variation

Description
Create a variation rule on a Dynamic Content block. Variations are
ordered; the first matching rule wins, otherwise `baseContent`
falls through.

v5 supports `Create` only — no PATCH / DELETE on variations. To
remove or reorder, recreate the parent Dynamic Content block.

Salesforce API
Account Engagement v5 REST API.

Endpoint
POST https://{pi-domain}/api/v5/objects/dynamic-content-variations

Headers
Authorization: Bearer <salesforce-access-token>
Pardot-Business-Unit-Id: <buId>
Content-Type: application/json
Accept: application/json

Body
{
  "dynamicContentId": 41,
  "comparison": "is",
  "operator": "EQUAL",
  "value1": "EMEA",
  "content": "<p>Bonjour!</p>"
}

Input (parameters)
operation (string, required) - "create" only.
dynamicContentId (number, required).
comparison (string, required) - "is" | "isNot" | "contains" |
  "doesNotContain" | "isBetween".
operator (string, required) - "EQUAL" | "NOT_EQUAL" |
  "GREATER_THAN" | "LESS_THAN" | "BETWEEN".
value1 (string, required).
value2 (string, optional) - Required for "isBetween" / "BETWEEN".
content (string, required) - HTML rendered when the rule matches.

Result
variation (object) - Created row.

Required scopes
- `pardot_api`.

Idempotency & limits
- Not idempotent.

Errors
- 20 invalid_input (operator / comparison mismatch),
  59 not_found (dynamicContentId).

Example request

manage_dynamic_content_variation({
  "operation": "create",
  "dynamicContentId": 41,
  "comparison": "is",
  "operator": "EQUAL",
  "value1": "EMEA",
  "content": "<p>Bonjour!</p>"
})

Example result
{ "variation": { "id": 92, "dynamicContentId": 41, "value1": "EMEA" } }
